
# 原程序功能说明：

1、已知目标是启动车子BaoMa，但在启动前需要先组装车子。

2、有三个部件需要组装，车架（CarFrame）、引擎（Engine）、电瓶（Battery）。引擎和电瓶要按工序先后组装到车架上。

3、组装完毕后，引擎（Engine）点火，车子启动。


# 新程序改造要求：

1、我们使用时，只需知道先组装（Assemble）、再启动（StartEngine），但我们完全不用去了解车子是如何组装的。

2、假如增加另一个品牌的车子BenChi，也是先组装（Assemble）、再启动（StartEngine），但与BaoMa不同的是，组装流程是先装电瓶再装引擎。

3、原程序的车架（CarFrame）、引擎（Engine）、电瓶（Battery）都用固定款型，但考虑到组装部件未来可能改用其他款型，比如组装BaoMa改用CAMEL电瓶。

4、为满足以上三点需求，请优化代码，体现面向对象封装性、灵活性、可拓展性等特点。假如以上需求无法做到同时兼顾，就按1、2、3优先顺序去考虑。
